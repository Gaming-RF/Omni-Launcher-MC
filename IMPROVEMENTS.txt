## Priority 2: Code Quality (High Impact)

### 2.1 Introduce Structured Error Type in Rust
Problem: 420+ occurrences of .map_err(|e| e.to_string()) across all command files. This loses error context, makes debugging hard, and returns unstructured strings to the frontend.
Files: All src-tauri/src/commands/*.rs, src-tauri/src/api/*.rs
Action:
- Create src-tauri/src/error.rs with an AppError enum (Db, Http, Io, Validation, NotFound, Auth, etc.)
- Implement Into<InvokeError> for Tauri integration
- Replace .map_err(|e| e.to_string()) with ? using From impls
- Frontend can then match on error codes for better UX

### 2.2 Remove .unwrap() Calls in Rust Commands
Problem: 4 .unwrap() calls in production code (library.rs:173, resource_packs.rs:101,106, graphics.rs:135). These will panic and crash the app if the values are None.
Files: src-tauri/src/commands/library.rs, resource_packs.rs, graphics.rs
Action: Replace with .ok_or_else(|| "reason".to_string())? or proper error handling.

### 2.3 Break Up Large Files
Problem: Several files exceed reasonable size limits:
- src-tauri/src/commands/minecraft.rs -- 972 lines
- src/lib/tauri.ts -- 1,202 lines (monolithic IPC wrapper)
- src/pages/Discover.tsx -- 733 lines
- src/pages/InstanceDetail.tsx -- 665 lines
- src-tauri/src/commands/loaders.rs -- 715 lines

Action:
- tauri.ts: Split into domain-specific modules (tauri/auth.ts, tauri/instances.ts, tauri/mods.ts, tauri/worlds.ts, etc.) with a barrel tauri/index.ts
- minecraft.rs: Extract modpack commands to commands/modpacks.rs, search to commands/search.rs
- Discover.tsx: Extract mod card grid, search filters, and mod detail modal into sub-components
- InstanceDetail.tsx: Already has tabs, but the mods tab logic is inline -- extract to ModsTab.tsx

### 2.4 Add React Error Boundary
Problem: Any uncaught React error will white-screen the entire app. No error boundary exists.
File: Create src/components/common/ErrorBoundary.tsx, wrap in App.tsx
Action: Class component error boundary with fallback UI and "reload" button.
